Triple's tech stack

Chanel - May 2023

Chanel is a frontend developer at Triple, a full service digital agency located in Alkmaar. She graduated from CMD at the Amsterdam University of Applied Sciences in 2018. In this talk, Chanel told us about Triple's tech stack and their motivation for using certain technologies.

Git

Git is an important tool at Triple. It's used for version control and makes it easier for multiple developers to collaborate on projects. The workflow in Git at Triple is as follows:

  1. Create a branch
  2. Commit & push
  3. Create a pull request
  4. Push to the main branch
  5. Deploy!

Npm

At Triple, when working on Node projects, developers are allowed to use packages from the Node Package Manager (npm). Using packages inside your project is something that should be thought over carefully because it could allow for vulnerabilities in your project. When using npm packages, it's important to check the following:

  • When was the last update?
  • How many downloads does it have?
  • How many contributors does it have?
  • How big is the package?
  • What are it's dependencies?

Vanilla, library or framework

When starting a project, one of the most important things to decide on is whether to use vanilla Javascript, or use a library or framework. Sometimes vanilla Javascript is the better option, but sometimes using a library or framework can help you prevent reinvent the wheel.

In the past, Triple used Angular a lot. Nowadays, they use React and Next.js. For projects where SEO (search engine optimilization) isn't imporant, React is a good option. When using React, content is rendered on the client which isn't good for SEO, but can give your website an "appy" feel. If SEO is important, Next.js is a better option. Next.js renders content on the server, either during build time or during runtime, which is better for SEO.

Triple is currently preparing to switch to Svelte. Svelte is a Javascript framework that is faster, lighter and easier to use than React. It's also easier to learn for developers who are new to Javascript frameworks since it's closer to vanilla Javacript.

Typescript

At Triple, Typescript is used a lot. Typescript is a superset of Javascript, which means that all valid Javascript is valid Typescript. Typescript adds enforces types in Javascript. Using Typescript helps allow developers to find bugs before they get to the browser. Below is some example code highlighting the differences between Javascript and Typescript. 

				
// Javascript
function add(a, b) {
	return a + b;
}

add(1, 2); // 3
add('1', '2'); // '12'

// Typescript
function add(a: number, b: number): number {
	return a + b;
}

add(1, 2); // 3
add('1', '2'); // Error: Argument of type 'string' is not assignable to parameter of type 'number'.

Of course, Typescript does a lot more than simple stuff like this. To read more about Typescript, check out their docs

.

CSS

At Triple, they use SASS, a CSS preprocessor. SASS is a superset of CSS, which means that all valid CSS is valid SASS. SASS adds features like variables, nesting, mixins, inheritance, etc. Below is some example code showing why SASS can be a useful too to write less CSS. 

				

@for $i from 1 through 3 {
	.item-#{$i} { width: 2em * $i; }
}


.item-1 {
	width: 2em;
}
.item-2 {
	width: 4em;
}
.item-3 {
	width: 6em;
}

Azure & Azure DevOps

Azure is a cloud computing service created by Microsoft. Azure DevOps is a service that provides developers with tools to help them with their development process. At Triple, they use Azure DevOps for their pipelines. A pipeline is a virtual machine that runs your code and checks it against the standards. Then builds it so it's ready for deployment. Triple uses three environments for testing:

  1. Development: for testing and developing
  2. Acceptance: for testing with the client
  3. Production: the live website
